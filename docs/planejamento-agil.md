# Planejamento Ágil - DuplaTrabalho

## Scrum

O desenvolvimento será realizado em três sprints semanais. Em cada início de sprint, a equipe seleciona itens priorizados do backlog. Ao final, será feita uma revisão das entregas e uma retrospectiva para identificar melhorias.

## Histórias de usuário

1. Como usuário, desejo cadastrar categorias para classificar os produtos.
2. Como usuário, desejo consultar categorias cadastradas para organizar o catálogo.
3. Como usuário, desejo cadastrar fornecedores para manter dados de contato atualizados.
4. Como usuário, desejo atualizar ou excluir fornecedores quando os dados mudarem.
5. Como usuário, desejo cadastrar produtos vinculados a uma categoria e a um fornecedor.
6. Como usuário, desejo consultar, alterar e excluir produtos para manter o estoque organizado.

## Quadro Kanban no GitHub Projects

Colunas necessárias: **Backlog**, **A Fazer**, **Em Desenvolvimento**, **Em Teste** e **Concluído**.

## Issues sugeridas

1. Criar estrutura inicial e documentação do projeto.
2. Modelar banco de dados e criar script SQL.
3. Implementar CRUD de categorias.
4. Implementar CRUD de fornecedores.
5. Implementar CRUD de produtos.
6. Testar navegação do menu e operações do sistema.
